# Preferências Globais do Usuário

## Identidade Visual e Temas
- Cores baseadas no domínio: #0f2557 (ML), #1a3a2a (Análise), #2d1a3d (Automação).
- Badges sempre usando a formatação `flat-square`.
- Priorizar estética tecnica (Dark/Teal).

## Tom de Voz (Copywriting)
- **LinkedIn:** Primeira pessoa ("Eu construí", "Chegamos a"), humano e com foco no problema de negócio real mitigado pela solução técnica. Sem uso excessivo de emojis / bullet points.
- **README:** Técnico e direto. Não usar "incrível", "poderoso", etc. Foco na stack e hero metric.
- **Sites:** Orientado a storytelling visual. Destaque forte para o código, hero metric interativo e visualização clara de insights.

## Regras Específicas
- Diferenciar os projetos de Ciência de Dados e EDA:
  - O projeto NumPy foca em tratamento de dados puro, vetorização matemática, eficiência básica.
  - O projeto Pandas foca em agregações de mercado, manipulações mais semânticas, geografia, mix de vendas.
- O Genesys_manager assume destaque como um Full-Stack robusto.
